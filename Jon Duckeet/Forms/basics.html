Form controls

  1. Adding text
    1. Text input - Used for a single line of text
    2. Password - Enter Passwords
    3. TextArea - For longer areas of text, such as messages and comments.
    4. Email - Enter Emails
  2. Making Choices
    1. Radio Button - For use when a user must select one of a number of options.
    2. Check Box - When a user can select and unselect one or more options.
    3. Drop-Down Boxes - When a user must pick one of a number of options from a list.
  3. Submitting Forms
    1. Submit Button - To submit data from your form to another web page.
    2. Image Buttons - Similar to submit buttons but they allow you to use an image.
  4. File Uploads button - Allows users to upload files (e.g. images) to a website.

action Every <'form> element requires an action attribute. Its value is the URL for the page on the server that will receive the information in the form when it is submitted

method Forms can be sent using one of two methods: get or post . With the get method, the values from the form are added to the end of the URL specified in the action attribute.
The get method is ideal for:

  • ● ● short forms (such as search boxes)
  • ● ● when you are just retrieving data from the web server (not sending information that should be added to or deleted from a database)


    With the post method the values are sent in what are known as HTTP headers. As a rule of thumb you should use the post method if your form:
  • ● ● allows users to upload a file
  • ● ● is very long
  • ● ● contains sensitive data (e.g. passwords)
  • ● ● adds information to, or deletes information from, a database If the method attribute is not used, the form data will be sent using the get method.
